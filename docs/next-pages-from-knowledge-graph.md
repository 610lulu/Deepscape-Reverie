# 深景迴梦网页后续页面规划：从知识图谱到空间叙事

本文档根据 `陆伊霖-小册子.pdf` 的内容整理，目标是明确当前网页从“知识图谱”之后应该继续补哪些页面、每一页承担什么叙事功能，以及如何把研究逻辑转译成可交互的网页体验。

当前网页已经具备梦境入口、空间节点、访谈摘录、设计策略图、知识图谱 iframe、放大镜/节点关系模式等基础。接下来不建议继续堆展板，而是从知识图谱开始，把“空间-事件-情绪”的关系梳理成三条可以被体验的叙事线。

## 一、从小册子提取出的核心叙事逻辑

小册子的设计方法可以压缩成四步：

```text
STEP 1 采访问到空间记忆点
-> STEP 2 通过知识图谱构建空间关系
-> STEP 3 提取空间光线特征，完成空间原型转译
-> STEP 4 通过梦核风格化转译记忆，生成空间效果
```

其中最适合继续接在网页知识图谱之后的是 STEP 2 到 STEP 4：

1. 知识图谱不是最终答案，而是叙事线索池。
2. 空间、事件、情绪需要被整理成几条“可走的路线”。
3. 每条路线再对应一种空间原型、光线策略和最终场景。
4. 最后回到线上学生叙事空间，让用户理解自己刚刚经历的是一套“记忆被重组”的过程。

## 二、知识图谱之后的网页主线

建议把后续页面组织成以下顺序：

```text
知识图谱模式切换
-> 从图谱中挖掘叙事线索
-> 三条叙事线选择
-> 叙事线详情页
-> 空间光线原型转译
-> 场景意象提取
-> 风格化生成过程
-> 线上学生叙事空间展示
-> 三个最终场景沉浸页
-> 结尾反思 / Thank You
```

这样用户不会停在“看一张复杂图谱”的状态，而是会继续被引导：我看见了关系，我选择一条线，我进入这条线的空间，我看见它如何被设计生成。

## 三、页面 1：从知识图谱挖掘叙事线索

### 页面目的

把当前知识图谱从“资料展示”变成“叙事入口”。用户先看到图谱，再被提示图谱中可以提取三条主要线索。

### 页面内容

标题建议：

```text
从知识图谱挖掘叙事线索
Mining Narrative Clues from the Knowledge Graph
```

页面核心结构：

1. 左侧保留知识图谱区域，支持当前的两种模式：
   `节点关系模式` 和 `放大镜模式`。
2. 右侧或下方出现三条线索卡片：
   `消失线`、`折叠线`、`变速线`。
3. 每张卡片显示对应的空间、事件、情绪关键词。
4. 用户点击其中一条线，进入该线的详情页。

### 三条叙事线

```text
LINE A 消失线
空间：码头 / 安来市商业街
事件：等船 / 迁徙 / 经过 / 停滞
情绪：失落 / 消失 / 停滞
解释：水运衰退带来空间功能消退，过去繁荣的商业街逐渐退场，只留下被访谈反复提及的集体记忆。

LINE B 折叠线
空间：学校 / 书院 / 幼儿园
事件：上学 / 成长 / 教育 / 代际记忆
情绪：怀旧 / 熟悉 / 陌生 / 尺度错位
解释：教育空间在不同年代反复出现，记忆以儿童视角和成人回望叠加，使教学空间成为多代身体经验的容器。

LINE C 变速线
空间：古村巷道 / 榕树公园 / 深井公园
事件：停留 / 玩耍 / 日常生活 / 慢节奏
情绪：熟悉 / 重逢 / 松弛 / 慢
解释：慢节奏给空间呼吸感，使身心回到自然节奏，在四季和生活日常中留下可持续的情绪体验。
```

## 四、页面 2：三条叙事线选择页

### 页面目的

让用户从研究图谱转入“可选择的叙事路径”。这一页可以做得像档案抽屉，也可以像三条发光路径。

### 交互建议

1. 三张大卡横向排列，分别对应三条线。
2. 悬停卡片时，背景图谱中相关节点和线条高亮。
3. 点击卡片后进入该叙事线详情。
4. 卡片不要写太多论文式说明，优先显示“空间 + 情绪 + 一句引文”。

### 文案草案

```text
消失线
码头与商业街的水运记忆正在退潮。
“以前的很大的船，现在都没有大船，只有小船。”

折叠线
学校、书院和幼儿园把不同年龄的身体经验折叠在一起。
“小学原来在村里面更深处的地方，现在搬到路边来了。”

变速线
公园、巷道和树影让日常生活慢下来。
“以前是农田，后来变成公园。”
```

## 五、页面 3：叙事线详情页

### 页面目的

每条叙事线需要一个详情页，解释它如何由空间、事件、情绪组合而来。

建议复用同一套页面模板，用户点击不同线时切换内容。

### 模板结构

```text
左侧：线索名称 / 核心引文 / 情绪状态
中间：空间节点序列
右侧：线索解读
底部：进入空间原型转译
```

### 详情内容

#### 消失线

空间序列：

```text
码头 -> 安来市商业街 -> 旧商业记忆
```

线索解读：

```text
水运的停靠引发空间消失。过去繁荣的商业街逐渐消退，空间向记忆迁移，只留下被谈论、被怀念、被重新观看的痕迹。
```

可转译为：

```text
入口空间 / 等待船笛 / 单侧玻璃透光 / 线性引导
```

#### 折叠线

空间序列：

```text
书院 -> 小学 -> 幼儿园
```

线索解读：

```text
教学空间在不同年代反复出现。儿童视角、成长经验和成人回望交叠，使空间不再是单一年代的房间，而是多层时间的折叠。
```

可转译为：

```text
学校廊道 / 教室室内 / 单侧门洞透光 / 单侧窗透光
```

#### 变速线

空间序列：

```text
古村巷道 -> 榕树公园 -> 深井公园
```

线索解读：

```text
慢节奏给空间呼吸感。树影、坡道、停留平台和自然光让用户从快速浏览转向慢速观看。
```

可转译为：

```text
室外坡道 / 榕树公园 / 自然天光 / 自然树影光模拟
```

## 六、页面 4：空间光线原型转译

### 页面目的

承接小册子 STEP 3，把三条叙事线转译成空间原型。这里是从“图谱关系”到“设计空间”的关键过渡。

### 页面标题

```text
空间光线原型转译
Spatial Light Archetype Translation
```

### 页面内容

建议做成三栏：

1. `消失线`：线性引导、单侧观看、延迟抵达、记忆边界。
2. `折叠线`：门洞重复、界面重叠、时间叠合、教学空间回声。
3. `变速线`：自然天光、树影筛光、节奏放缓、停留平台。

每栏包含：

```text
原始空间
-> 光线特征
-> 空间原型
-> 用户体验
```

### 可用文本

```text
光线不只是照明条件，而是组织观看、行走、停留与时间感知的媒介。网页可以将光线作为叙事线索，让用户看见记忆如何从访谈语言转译为空间体验。
```

## 七、页面 5：场景中意象物的提取

### 页面目的

将“光、物、影”三种意象元素展示出来，作为生成空间场景的素材库。

### 页面结构

```text
01 光的提取
原始元素：教室条形灯 / 线性光带
转译结果：连续线性灯带、方向性引导、自然光天窗

02 物的提取
原始元素：饮水机 / 门牌 / 路牌 / 记忆锚点
转译结果：可识别的场景锚点、节点标识、路径记忆装置

03 影的提取
原始元素：树枝阴影 / 建筑投影 / 构件阴影
转译结果：参数化树影、边界阴影、可穿越的空间网架
```

### 交互建议

1. 做成三个可切换标签页：`光`、`物`、`影`。
2. 每个标签页从左到右显示：原始照片、提取图形、空间转译。
3. 点击“生成空间片段”后进入下一页。

## 八、页面 6：梦核风格化生成过程

### 页面目的

展示 STEP 4：如何用 ComfyUI / GPT Image 2 等方法把空间原型变成最终视觉氛围。

### 页面标题

```text
通过梦核风格化转译记忆
Translate Memory Through Liminal Aesthetic
```

### 页面内容

建议做成“生成流程时间线”：

```text
01 保留原始建筑关系
严格保留建筑几何、结构、比例、材料分布、视角与空间关系。

02 注入视觉风格
只进行视觉氛围转译：红色材质、玻璃界面、高对比电影光、空旷感、低饱和环境。

03 筛选场景结果
保留能准确表达三条叙事线的场景，形成最终漫游节点。
```

### 可加入素材

1. ComfyUI 节点截图。
2. 风格化效果试验 1.0。
3. 风格化效果试验 2.0。
4. 最终三张场景图。

## 九、页面 7：线上学生叙事空间的线上体验

### 页面目的

这里对应小册子第 30-31 页：网页本身也是成果的一部分，不只是展示载体。应当明确告诉用户：他们正在体验的是一个线上学生叙事空间。

### 页面标题

```text
线上学生叙事空间的线上体验
Online Experience of Student Narrative Space
```

### 页面结构

建议用多屏倾斜网页截图拼贴，但不要只当海报。可以做成可点击的“成果索引”：

1. `梦境入口`：空间记忆卡片。
2. `记忆解释`：梦境片段与研究解释。
3. `设计策略`：小册子展板翻阅。
4. `知识图谱`：节点关系与放大镜模式。
5. `访谈记录`：口述材料滚动。
6. `空间漫游`：进入最终场景。

每个截图点击后跳回网页对应章节。

## 十、页面 8：三个最终场景沉浸页

### 页面目的

用最终场景图作为结尾高潮，让用户进入由叙事线生成的空间。

### 三个场景

```text
童影穿廊
对应：折叠线
关键词：学校廊道、儿童视角、单侧透光、成长回声
文案：“可以的话，带我回到那一天。”

书声未远
对应：折叠线 / 教室记忆
关键词：教室、桌椅阵列、红色顶面、暂停的时间
文案：书声没有真正离开，只是被空间压低了声音。

落日嬉游
对应：变速线
关键词：红色坡道、日光、儿童停留、慢节奏
文案：慢下来的时候，场地重新开始保存身体的影子。
```

### 交互建议

1. 三张图做成全屏横向滑动。
2. 每张图只保留一句大文案和一组小标签。
3. 鼠标移动时产生轻微视差，让图像不像静态展板。
4. 页面底部保留“回到知识图谱”与“查看设计生成过程”两个入口。

## 十一、页面 9：结尾反思页

### 页面目的

让用户完成体验后回到研究问题：如何在媒介碎片化与线性叙事的双重困境中，重构记忆连续性与空间参与感？

### 页面文案草案

```text
深井村不是被完整复原，而是被重新感知。

访谈让记忆成为线索，知识图谱让线索显出关系，光线和空间原型让关系变成可以行走的经验。网页中的每一次点击、悬停和进入，都是一次对“集体记忆如何被重新组织”的尝试。
```

### 结尾按钮

```text
重新进入梦境
回看知识图谱
查看完整设计流程
```

## 十二、建议开发顺序

### P0：先完成叙事线页面

1. 在知识图谱下方新增“三条叙事线”选择区。
2. 点击 `消失线 / 折叠线 / 变速线` 后显示详情。
3. 每条线绑定空间、事件、情绪、引文和空间原型。

### P1：补空间转译页面

1. 新增 `空间光线原型转译` 页面。
2. 新增 `光 / 物 / 影` 意象提取页面。
3. 把当前设计策略图片和新提取逻辑连接起来。

### P2：补风格化生成页面

1. 展示 ComfyUI / GPT Image 2 的生成流程。
2. 展示风格化效果试验 1.0 / 2.0。
3. 解释如何从原始空间保留几何关系，再进行梦核风格化。

### P3：补最终成果与反思

1. 做线上学生叙事空间成果索引。
2. 做三个最终场景沉浸页。
3. 做结尾反思页，回扣研究问题。

## 十四、一个更顺的网页章节命名

建议后续导航不要再用太多论文式词汇，可以用下面这套：

```text
Dream      梦境入口
Archive    访谈档案
Graph      知识图谱
Lines      叙事线索
Light      光线原型
Elements   意象提取
Generate   风格化生成
Scene      空间漫游
Final      最终呈现
```

这套命名比纯 `STEP1 / STEP2 / STEP3` 更适合网页，也能保留小册子的研究逻辑。

## 十五、下一次实际开发建议

下一次最值得先写的不是最终场景，而是 `Lines` 页面。因为它会把现在已经有的知识图谱真正转化为后续网页的叙事发动机。

建议最小可开发版本：

```text
1. 在 knowledgeGraph 后新增 section#lineMining
2. 放三张 narrative-line-card
3. 每张卡点击后更新 section#lineDetail
4. lineDetail 显示：空间序列、事件序列、情绪序列、访谈引文、空间原型入口
5. “进入空间光线转译”按钮跳到 section#lightArchetype
```

这样页面会从“看图谱”自然过渡到“走叙事线”，也和小册子的设计方法完全对上。
