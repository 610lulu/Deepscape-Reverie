## 视觉小说改版方向

根据视觉小说资料，当前项目不应只被理解为“网页展示”，而应改成：

```text
视觉小说 = 叙事 + 视觉/听觉演出 + 交互性
```

对应到 `Deepscape Reverie`：

1. **叙事 Narrative：游戏的灵魂**
   玩家不是在阅读一份设计说明，而是在梦里回到深井村。叙事决定玩家感受到什么：迷路、等待、陌生、怀旧、重逢、尺度错位。
2. **演出 Presentation：游戏的血肉**
   场景渲染图、空间影像、字幕、转场、BGM、环境声、卡片 UI 一起传递信息，降低纯文字对玩家的认知负担。
3. **交互 Interagency：游戏的骨架**
   玩家通过选择空间、收集线索、拾取字条、拼合关系、解锁原型，产生“我的选择改变了世界显影方式”的沉浸感。

### 视觉小说主线

现在的项目可以升级成一条视觉小说式主线：

```text
梦境入口
-> 选择空间卡
-> 梦境片段弹窗
-> 梦醒后的物件清单
-> 收集访谈字条
-> 进入空间场景影像
-> 拼合空间 / 事件 / 情绪
-> 解锁六个空间原型
-> 原型风格化转译
-> 空间场景拼贴
-> 完成“记忆博物馆”空间
```

这个结构里，研究文本不再是第一层内容，而是玩家通关后的解释层。用户先经历梦，再收集证据，最后才看见设计逻辑。

### 剧本结构

视觉小说需要先拆成“主线 / 分支点 / 个人线”：

1. **主线：所有玩家必读**
   2026 年 6 月，“我”做了一个梦，醒来后桌上多了六件物件。玩家必须收集线索，拼合出深井村的空间记忆，并让最终博物馆显影。
2. **分支点：玩家做选择的地方**
   首页五张空间卡、六件物件、随机空间探索、访谈字条、关系拼合，都是玩家可以选择顺序的地方。
3. **个人线：每个空间的专属剧情**
   码头、商业街、书院、公园、小学、幼儿园各自有一段梦境片段、一张访谈字条、一条空间影像、一种空间原型。

对话和文本不必按时间顺序写。更适合采用“梦境闪回”的写法：先给高情绪片段，再通过字条、图谱和空间原型补解释。

### 美术与听觉资源

后续素材应按视觉小说资源类型整理：

1. **背景 BG**
   深井村现场照片、空间渲染图、拼贴后的博物馆场景。
2. **关键插画 CG**
   关系拼合成功、空间原型解锁、博物馆显影等关键节点画面。
3. **UI 界面**
   对话框、物件卡、字条、选择按钮、已收集档案、章节进度。
4. **听觉资源**
   低频环境声、水声、远处课堂声、脚步、纸张声、梦境 BGM。当前页面还没有声音，这是视觉小说感最值得补的一层。

## 游戏叙事试做

已按新的游戏叙事链调整为：

```text
梦醒 -> 物件清单 -> 随机空间探索 -> 访谈字条 -> 关系拼合 -> 空间原型解锁 -> 单图空间漫游 -> 最终图谱
```

当前页面对应关系：

1. `dream`：玩家先在梦里收集五个空间记忆，集满后才会“梦醒”。首页已去掉 `01 安来市商业街`，商业街改留到梦醒后的证据层出现。
2. `wakeRoom`：梦醒后进入一整页物件清单，六张卡片成为独立任务页，而不是附属面板。
3. 点击任意物件后，页面自动进入对应访谈字条文本层，显示引文、地点线索与空间原型。
4. `spaceExplore`：六件物件收齐后，进入随机空间探索，不再按论文顺序阅读。
5. `spaceExplore` 里的访谈字条需要逐张捡起，收齐六张后才允许进入关系拼合。
6. `relationPuzzle`：把空间、事件、情绪放回同一条关系链，六条关系都拼上后，空间原型解锁。
7. `prototypeGallery`：接入三张场景渲染视觉氛围图，作为玩家完成拼合后看到的“空间谜底”。
8. `sceneWalk`：把一张红色楼梯渲染图做成 2.5D 单图空间漫游，玩家通过热点和前进/后退按钮在图中切换视点。
9. `knowledgeGraph`：最终图谱作为“解释层”出现，研究文本变成玩家完成探索后的参考答案。
10. 顶部导航已调整为 `Dream / Tasks / Explore / Match / Scene / Walk / Graph`，访谈区的“查看任务清单”也跳转到任务桌。

下一步可以把当前“点击放入关系链”升级为真正的拖拽归类：玩家需要把访谈字条拖到正确的空间、事件、情绪槽位，错配时给出轻微反馈。

### 场景渲染图接入策略

三张新图不建议只放在项目介绍或普通画廊里。它们最适合作为“关系拼合完成后的空间显影”，承担从研究线索到设计成果的转场：

1. `scene-render-01.jpg`：红色楼梯、光廊、树影，适合表达“等待的边界 / 上行路径 / 时间被拉长”。
2. `scene-render-02.jpg`：桌椅、红色顶面、悬置灯带，适合表达“童年回声场 / 教室记忆 / 被暂停的日常”。
3. `scene-render-03.jpg`：玻璃夹层、楼梯、儿童尺度，适合表达“关系重组 / 透明剖面 / 空间原型显影”。

网页风格上，应把这三张图当成主视觉基因：更宽的电影画幅、更强的红色材质、更透明的玻璃层、更明确的光线切割。前面的任务界面保持暗色、档案感和线索感；完成拼合后再突然进入高亮的红色场景，会更有“梦醒后看见答案”的节奏。

### 单张图空间漫游策略

`000.png` 已转成 `assets/scene-walk-01.jpg`，用于 `sceneWalk`。它目前采用的是“单图 2.5D 漫游”，不是完整 3D 建模或 360 全景：

1. 背景仍然是一张渲染图，但通过 `scale / translate / transform-origin` 做镜头推进和视线偏移。
2. 热点绑定在图像空间中：入口、台阶、树影、柱列分别对应四个梦境视点。
3. 每个视点都有一段视觉小说式旁白，让“移动”不只是换图，而是进入空间记忆。
4. 这一版适合快速展示“人在图里的空间里逛”的感觉；后续如果需要更强沉浸，可以升级为多张视角图、景深分层、鼠标视差、脚步声，或真正的 Three.js 走廊场景。
5. 当前最佳叙事位置是：空间原型解锁后，玩家不立刻看论文图谱，而是先进入图里的红色光廊走一小段，再进入最终图谱。

### 梦境弹窗写法

首页五张空间卡点击后，不再使用普通说明文字，而是进入“梦境片段”：

1. 第一层是梦里的主观经验：用“我走进 / 我站在 / 我没有看见”等第一人称，让用户像进入场景。
2. 第二层保留 `Space / Event / Emotion`，但它们只是标签，不抢走梦境叙事。
3. 第三层是 `Dream Clue`，把研究结论压缩成一句谜底线索，例如“商业街显影为褪色的日常舞台”。
4. 图片优先使用红色场景渲染图，而不是现场照片。现场照片可以留给后面的档案或图谱解释层。
5. 文体避免“某空间承载了某功能”这类论文句式，改成可感知的空间动作：上行、停留、隔着玻璃观看、走不到尽头、桌椅排队、尺度错位。

这种处理能让“梦境入口”和后面的“访谈证据 / 知识图谱”形成层次：先感受，再解谜，最后解释。

## 5. 建议新增的网站章节结构

当前页面已经开始从“作品展示顺序”转向“游戏叙事顺序”。建议后续继续按这条主链组织：

```text
01 Dream Entrance / 梦境入口
   先做梦，点击六个空间节点，收集基础记忆。

02 Waking Desk / 物件清单
   梦醒后出现六件物件，物件只提供谜面，不直接解释研究结论。

03 Random Space / 随机空间探索
   随机进入码头、商业街、书院、公园、小学、幼儿园，打破论文式线性阅读。

04 Interview Slips / 访谈字条
   玩家捡起访谈字条，逐步获得空间、事件、情绪线索。

05 Relation Assembly / 关系拼合
   把字条放回“空间 - 事件 - 情绪”的链条里。

06 Prototype Unlock / 空间原型解锁
   六条关系拼合后，解锁六个设计原型。

07 Final Graph / 最终图谱
   图谱作为谜底层出现，解释前面收集到的线索。

08 Archive / 参考档案
   项目介绍、设计策略、互动电影、访谈记录作为通关后的研究档案。

09 Final Space / 最终空间
   最终建筑/博物馆/空间叙事成果
```

如果时间有限，优先强化 `05 Relation Assembly` 和 `09 Final Space`，因为前者决定它像不像游戏，后者决定项目成果是否完整。

## 7. 可以直接改进的页面细节

1. `relationPuzzle` 现在是点击式拼合，下一步应改成拖拽式归类。
2. 访谈字条目前只显示“已收入档案”，可以增加已收集列表或小型档案夹。

7. 弹窗关闭按钮已有，但可以补 `Tab` 焦点限制和关闭后焦点返回。
8. 大图资源总量较高，正式展示前应压缩并检查加载速度。

## 9. 阶段性验收标准

下一版完成后，建议用这些问题检查：

1. 不看 PDF，用户是否知道项目发生在深井村？
2. 用户是否能理解六个节点来自访谈和空间变化，而不是随意选的地点？
3. 用户是否能按“梦醒 -> 物件清单 -> 随机空间探索 -> 访谈字条 -> 关系拼合 -> 空间原型解锁 -> 单图空间漫游 -> 最终图谱”走完一轮？
4. 用户是否知道自己下一步要收集什么、拼合什么、解锁什么？
5. 移动端是否能自然收集全部五个梦境节点和六张访谈字条？
6. 点击电影路径时是否不会出现空视频或困惑提示？
7. 知识图谱是否像最终谜底，而不是孤立嵌入？
8. 页面是否讲清楚“梦境体验 -> 调研证据 -> 方法转译 -> 空间成果”？
9. 图片和视频加载是否不会明显拖慢展示节奏？
10. 视觉小说三要素是否成立：叙事让人想继续读，演出让人进入场景，交互让人觉得选择有意义？
11. 用户是否能分清三层内容：梦境层、线索层、解释层？
12. 每个空间是否都有自己的专属剧情、场景图、线索物和空间原型？

## 10. 最短可交付版本

如果只剩很短时间，建议先完成这 6 件事：

1. 保留当前游戏主链，并确保完整通关无断点。
2. 先做视觉小说底部对话框，把首页五个梦境片段变成逐句推进。
3. 把关系拼合升级为拖拽式归类。
4. 给六个空间原型补 1 张总览图或 6 张小型概念图。
5. 新增一个简短 `Final Space` 章节，哪怕先用 2-3 张关键图。
6. 压缩首屏以外大图，保证展示时加载稳定。

完成这 6 件后，网站就会从“氛围很强的互动首页”升级为“能玩、能读、也能讲清楚研究逻辑和设计成果的互动叙事作品”。
